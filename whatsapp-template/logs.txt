# 🧠 WhatsApp Template: `job_assigned` (SYSTEM DESIGN EXPLAINED)

## 📌 What we built

We created a **WhatsApp Utility template** called:

```text id="tpl1"
job_assigned
```

This template is used to:
👉 Notify technicians about a new job assignment
👉 Allow them to respond instantly (ACCEPT / REJECT)

---

# 🎯 Why we designed it this way

## 1. Utility classification (important for delivery)

We structured it as a **Utility message**, not marketing.

✔ Reason:

* Job dispatch = transactional system message
* Higher delivery rate
* Lower risk of rejection or throttling

---

## 2. Structured data format (machine-friendly)

### Body:

```text id="body1"
New job assigned: {{1}}

Trade: {{2}}
Location: {{3}}

Job details:
{{4}}

Please respond to accept or reject this job.
```

### Why:

* Each variable maps directly to DB fields
* Easy parsing in logs and analytics
* Consistent formatting for technicians

---

## 3. Variables mapping (critical for OpenCode logic)

| Variable | Meaning             | Example         |
| -------- | ------------------- | --------------- |
| {{1}}    | Job ID              | JOB-10492       |
| {{2}}    | Trade               | Electrician     |
| {{3}}    | Location            | Islamabad       |
| {{4}}    | Problem description | Fan not working |

---

## 4. Quick Reply Buttons (core interaction design)

We added:

### Button 1:

```text id="btn1"
ACCEPT
```

### Button 2:

```text id="btn2"
REJECT
```

---

### Why buttons instead of text?

✔ Faster response (1 tap vs typing)
✔ Higher acceptance rate
✔ Easier webhook handling
✔ Cleaner automation flow

# ⚙️ HOW OPEN CODE SHOULD IMPLEMENT THIS

## 1. Sending template message

Your backend should call WhatsApp Cloud API like:

```js id="api1"
{
  "messaging_product": "whatsapp",
  "to": technician_phone,
  "type": "template",
  "template": {
    "name": "job_assigned",
    "language": { "code": "en" },
    "components": [
      {
        "type": "body",
        "parameters": [
          { "type": "text", "text": jobId },
          { "type": "text", "text": trade },
          { "type": "text", "text": location },
          { "type": "text", "text": summary }
        ]
      }
    ]
  }
}
```

---

## 2. Button response handling (VERY IMPORTANT)

When technician clicks:

### 👉 ACCEPT

Webhook payload:

```text id="w1"
button_payload = "ACCEPT"
```

### Logic:

```js id="logic1"
if (payload === "ACCEPT") {
  assignJobToTechnician();
  sendConfirmation();
  stopBroadcasting();
}
```

---

### 👉 REJECT

```js id="logic2"
if (payload === "REJECT") {
  markJobAsRejected();
  findNextTechnician();
}
```
